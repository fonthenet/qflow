session (2026-04-23):

           Both builds produced identical First Load JS numbers, confirming
           that withSentryConfig webpack plugin injects Sentry SDK regardless
           of the sentry.client.config.ts file once instrumentation-client.ts
           exists. Build B is the canonical AFTER baseline.

==============================================================================
ROOT CAUSE ANALYSIS
==============================================================================

The 221→222 KB shared bundle is dominated by:

  128 KB gz: Next.js App Router internals (router, prefetch, navigation,
             server actions transport) combined with Sentry's auto-instrumentation
             hooks injected by withSentryConfig webpack plugin. The Sentry portion
             (captureException, BrowserTracing, Replay core) is estimated at
             60–80 KB gz of this chunk. It cannot be excluded without disabling
             the webpack plugin entirely.

  54 KB gz:  React DOM — not reducible.

  36 KB gz:  Shared app code (locale, auth, UI primitives).

The booking page (/book/[officeSlug]) at 400 KB is 400 KB = 222 KB shared +
178 KB page-specific. The page chunk itself is only 2 KB — the 178 KB difference
comes from Supabase client SDK and booking-form component dependencies pulled into
shared route-group chunks, not from the page itself.
